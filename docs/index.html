Evaluation of the Fréchet Radar Distance for the Quality Assessment of Synthetic Radar Data

Nicolas Zander
OTH Regensburg - Aumovio
2026
PDF Code

TL;DR: FRD is a FID-inspired distance for radar point clouds using random projections as a feature extractor (no domain-pretrained network required). Validated FRD in controlled pseudo-radar scenarios and compared it to an approximated log-likelihood reference. Applied FRD to evaluate U-Net diffusion models for radar data generation and analyzed training dynamics and model capacity. Key finding: FRD is consistent, reproducible, and behaves as expected in controlled settings; diffusion models can reproduce radar-like structure under idealized conditions.


Motivation

Evaluating synthetic radar data is difficult because common metrics (e.g., FID) depend on domain-specific feature extractors that do not transfer well to radar. Radar point clouds are sparse, noisy, and physically constrained, so we need a robust, distribution-based metric that does not rely on pretrained image models.

Method Overview

Pipeline: point clouds > discretization > random projections > Gaussian statistics > Fréchet distance.

Discretization

Convert sparse radar point clouds into a fixed representation suitable for statistical comparison.

Random Projections

Use random projections as a feature extractor to avoid domain-pretrained networks.

FRD

Compute mean/covariance in the projected space and compare distributions via Fréchet distance.

Experimental Setup

  • Controlled data: parameterized pseudo-radar generators (targeted distribution shifts).
  • Reference metric: approximated log-likelihood to quantify true distribution similarity.
  • Protocol: multiple seeds/runs, report mean ± variability; fixed projection settings for fair comparison.

Results: FRD Validation (RQ1)

  • Consistency: FRD remains low/stable for identical distributions across seeds.
  • Sensitivity: FRD increases with targeted parameter changes (expected ordering).
  • Reference alignment: FRD correlates with approximated log-likelihood in controlled scenarios.

Limitations

  • Validation uses synthetic / pseudo-radar distributions; transfer to real radar data is future work.
  • Primarily distribution-based evaluation; no task-based downstream validation included.
  • FRD depends on representation choices (e.g., discretization, projection count/dim).
